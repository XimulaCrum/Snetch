# Snetch

[English](README.md) | [한국어](README.ko.md)

대기열 중심으로 다듬은 안드로이드 영상/오디오 다운로더입니다.

Snetch는 Seal이 갖고 있던 `yt-dlp` 기반의 강점을 그대로 가져오되, 실제로 오래 써볼 때 거슬리던 부분을 더 정리하는 쪽으로 손본 포크입니다. 링크를 여러 개 계속 넣는 상황에서도 덜 어수선하고, 지금 뭐가 돌고 있고 뭐가 기다리는지 더 잘 보이게 만드는 데 초점을 두고 있습니다.

## 왜 만들었나

안드로이드용 다운로더는 처음엔 빨라 보여도 조금만 무거워지면 금방 지저분해지는 경우가 많습니다.

- 여러 작업이 한꺼번에 저장소와 CPU를 건드립니다
- 대기 중인 작업과 진행 중인 작업이 한눈에 안 들어옵니다
- 후처리까지 겹치면 약한 기기에서 체감이 크게 무너집니다
- 데이터 환경인데도 의도치 않게 다운로드가 시작되기도 합니다

Snetch는 이런 지점을 줄이기 위해 만든 포크입니다.

## 지금 손보고 있는 방향

현재는 아래 같은 개선에 힘을 주고 있습니다.

- 병렬로 한꺼번에 밀기보다 순차적으로 처리하기
- 저장해 둔 프리셋으로 링크를 바로 대기열에 올리기
- 플레이리스트 링크를 개별 작업으로 풀어서 대기열에 넣기
- Wi-Fi 조건이 맞을 때까지 작업을 대기시키기
- 안드로이드 임시 파일과 후처리 경로를 좀 더 보수적으로 다루기
- 대기 상태와 실패 이유를 더 읽기 쉽게 보여주기

기능을 덜어내겠다는 뜻은 아닙니다. 기본 동작을 먼저 단단하게 만들겠다는 쪽에 가깝습니다.

## 주요 기능

- [`yt-dlp`](https://github.com/yt-dlp/yt-dlp) 지원 사이트의 영상/오디오 다운로드
- 대기열 중심 작업 관리
- 기본 순차 다운로드
- 오디오/비디오 프리셋 저장 후 반복 다운로드 자동 처리
- 플레이리스트 링크 일괄 추가
- Wi-Fi only 기반 대기 처리
- 포맷 선택, 자막, 썸네일, 메타데이터, 커스텀 명령 템플릿 지원
- Material 3 기반 안드로이드 UI

## 최근 정리한 것

- upstream 앱과 공존 설치가 가능하도록 `applicationId` 분리
- 포크 전용 브랜딩과 런처 아이콘 정리
- 링크 추가 시 정보부터 읽고 대기열에 넣는 동작 보강
- 플레이리스트 항목 개별 대기열 추가
- 더 또렷한 대기 상태 문구
- Cloudflare, 로그인 필요, 쿠키 필요, 파일 접근 문제 같은 흔한 실패를 좀 더 직접적으로 안내

## 릴리즈 서명

공개 배포용 APK는 전용 서명 키를 쓰는 게 맞습니다.

- 샘플 설정 파일: [`keystore.properties.example`](keystore.properties.example)
- 설정 안내: [`docs/release-signing.md`](docs/release-signing.md)

지금은 커스텀 키가 없더라도 테스트용 릴리즈 APK가 설치되도록 debug 키로 fallback 하게 되어 있습니다. 로컬 테스트에는 편하지만, 장기적으로는 전용 릴리즈 키를 쓰는 편이 맞습니다.

## 설치

배포는 GitHub Releases 기준으로 진행합니다.

대부분의 최신 안드로이드 기기에는 아래 APK를 권장합니다.

- `arm64-v8a`

## 상태

Snetch는 계속 손보고 있는 포크입니다. 내부 소스 구조는 아직 upstream 흔적이 남아 있는 부분이 있지만, 설치 패키지와 공개 브랜딩은 이미 분리해 둔 상태입니다.

## 크레딧

Snetch는 [Seal](https://github.com/JunkFood02/Seal)에서 출발한 포크입니다.

아래 프로젝트들의 작업 위에 올라가 있습니다.

- [`yt-dlp`](https://github.com/yt-dlp/yt-dlp)
- [`youtubedl-android`](https://github.com/yausername/youtubedl-android)
- [`aria2`](https://github.com/aria2/aria2)
- [`mutagen`](https://github.com/quodlibet/mutagen)

- 재현 순서
- 로그나 스크린샷
