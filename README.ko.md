# Snetch

[English](README.md) | [한국어](README.ko.md)

안드로이드용 영상/오디오 다운로더.

Snetch는 `yt-dlp` 기반의 강력함은 유지하되, 실제 사용감은 조금 다른 방향으로 다듬고 있는 포크입니다. 핵심은 간단합니다. 여러 작업을 한꺼번에 몰아치는 대신, 링크를 쌓아두고 차례대로 처리하게 만들자는 겁니다.

빨라 보이는 것보다 중요한 건, 오래 켜두고 써도 덜 어수선한가입니다. 대기 중인 작업이 뭔지, 지금 뭐가 내려받는 중인지, 왜 멈췄는지가 눈에 들어와야 하고, 기기가 괜히 후처리와 저장소 접근 때문에 버벅이지 않아야 합니다. Snetch는 그런 쪽에 더 신경 쓰고 있습니다.

## 왜 만들었나

안드로이드용 다운로더는 처음엔 시원해 보여도, 작업이 몇 개만 쌓이면 금방 난잡해지는 경우가 많습니다.

- 여러 다운로드가 동시에 저장소와 CPU를 건드림
- 대기 중인 작업과 진행 중인 작업이 뒤섞여 보임
- 후처리까지 겹치면 약한 기기에서 체감이 확 나빠짐
- 셀룰러에서 원치 않게 다운로드가 시작되기도 함

Snetch는 이런 문제를 피하려고 만든 포크입니다.

이 앱이 우선하는 건 대체로 이런 것들입니다.

- 병렬로 마구 돌리기보다 순서대로 처리하기
- 백그라운드에서 조용히 꼬이기보다 상태가 잘 보이기
- 실수로 데이터를 쓰기보다 Wi-Fi 조건을 존중하기
- 기능을 늘리는 것보다 일단 동작을 단단하게 만들기

## 지금 집중하는 것

현재는 몇 가지를 우선순위로 잡고 손보고 있습니다.

- 링크 여러 개를 넣어도 덜 지저분한 대기열 동작
- 비과금 네트워크가 잡힐 때까지 작업을 멈춰두는 Wi-Fi only 처리
- 다운로드와 후처리 중 Android 임시 파일 경로 다듬기
- 공격적인 동시 작업에 약한 기기에서도 버티는 기본 동작 만들기

즉, 기능을 과하게 늘리기보다 기본 동작을 먼저 정리하는 쪽입니다.

## 기능

- [`yt-dlp`](https://github.com/yt-dlp/yt-dlp)가 지원하는 사이트의 영상/오디오 다운로드
- 링크를 쌓아두고 관리하는 대기열 중심 방식
- 기기 부담을 줄이는 순차 다운로드
- 비과금 네트워크에서만 시작되도록 하는 Wi-Fi only 동작
- 포맷 선택, 오디오 추출, 자막, 썸네일, 메타데이터 지원
- 고급 사용자를 위한 사용자 지정 `yt-dlp` 명령 템플릿
- Material 3 기반 Android UI

## 성향

Snetch는 "한 번에 다 받아버리자" 쪽보다는, "일단 줄 세우고 하나씩 끝내자" 쪽에 더 가깝습니다.

이런 사용자에게 잘 맞습니다.

- 대기 중인 작업을 한눈에 보고 싶은 사람
- 폰이나 태블릿에서 안정적으로 돌아가길 바라는 사람
- 백그라운드에서 뭔가 꼬이는 일을 줄이고 싶은 사람
- 강력하지만 덜 산만한 도구를 선호하는 사람

다운로더가 조용히 제 할 일만 잘했으면 좋겠다는 쪽이라면, 아마 취향에 맞을 겁니다.

## 설치

배포는 GitHub Releases 기준으로 진행합니다.

대부분의 최신 안드로이드 기기에는 아래 APK를 권장합니다.

- `arm64-v8a`

예정된 릴리즈 채널:

- 일상적으로 쓰는 안정 빌드
- 새 변경을 먼저 시험해보는 프리뷰 빌드

## 상태

Snetch는 아직 포크 분리 작업이 진행 중입니다. 내부 소스 패키지나 구조에는 upstream 흔적이 남아 있는 부분이 있지만, 설치 패키지명과 공개 브랜딩은 별도 앱으로 정리해 두었습니다.

지금 시점에서는 대략 이런 상태라고 보면 됩니다.

- 브랜딩 정리 진행 중
- 패키지와 릴리즈 자산 정리 진행 중
- 에뮬레이터와 실기기 양쪽에서 계속 검증 중

## 로드맵

- 포크 전용 브랜딩과 릴리즈 자산 정리
- 독립 배포를 위한 패키지 정체성 추가 정리
- 대기 상태와 진행 상태가 더 잘 보이도록 UI 손보기
- Android 저장소 및 후처리 관련 동작 계속 안정화
- 좀 더 다듬어진 공개 릴리즈 준비

## 크레딧

Snetch는 [Seal](https://github.com/JunkFood02/Seal)에서 출발한 포크입니다.

그리고 아래 프로젝트들의 작업 위에 올라가 있습니다.

- [`yt-dlp`](https://github.com/yt-dlp/yt-dlp)
- [`youtubedl-android`](https://github.com/yausername/youtubedl-android)
- [`aria2`](https://github.com/aria2/aria2)
- [`mutagen`](https://github.com/quodlibet/mutagen)

기반이 되는 오픈소스 프로젝트와 유지보수자들의 작업이 없으면 이 포크도 존재할 수 없습니다.

## 기여

이슈를 열 때는 아래 정보를 같이 적어주면 큰 도움이 됩니다.

- 기기 모델명
- Android 버전
- 문제가 Wi-Fi에서 났는지 셀룰러에서 났는지
- 대략적인 재현 단계
- 관련 로그

두루뭉술한 불만보다, 짧고 재현 가능한 보고가 훨씬 처리하기 쉽습니다.
