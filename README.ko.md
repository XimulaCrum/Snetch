# Snetch

[English](README.md) | [한국어](README.ko.md)

안드로이드용 큐 중심 비디오/오디오 다운로더.

Snetch는 `yt-dlp` 기반의 강력함은 유지하면서도, 더 차분하고 예측 가능한 다운로드 흐름을 목표로 만든 안드로이드 다운로더입니다. 여러 작업을 한꺼번에 터뜨리는 방식보다, 링크를 자연스럽게 쌓아두고 순서대로 처리하는 쪽에 더 집중합니다.

무턱대고 빠르기만 한 다운로드 앱이 아니라, 실제 기기에서 오래 써도 덜 어수선하고 덜 피곤한 앱을 지향합니다. 링크를 계속 추가해도 큐가 읽기 쉬워야 하고, 다운로드 상태가 한눈에 보여야 하며, 기기가 감당하기 어려운 과한 병렬 처리로 흐트러지지 않아야 합니다.

## 왜 Snetch인가

많은 안드로이드 다운로더는 처음엔 빨라 보여도, 작업이 쌓이기 시작하면 금방 산만해집니다.

- 여러 작업이 동시에 저장소와 CPU를 두드림
- 큐가 금방 읽기 어려워짐
- 무거운 후처리 때문에 약한 기기에서 버벅임이 커짐
- 셀룰러 환경에서도 원치 않게 다운로드가 시작됨

Snetch는 이런 불편함에 대한 답으로 만들어졌습니다.

Snetch가 우선하는 건 다음과 같습니다.

- 무질서한 동시 실행보다 순차 다운로드
- 보이지 않는 백그라운드 난전보다 읽기 쉬운 큐
- 실수로 데이터가 나가는 일보다 Wi-Fi를 고려한 시작 조건
- 기능 과잉보다 실제 사용 안정성

## 현재 방향

Snetch는 지금 몇 가지 개선에 집중하고 있습니다.

- 여러 링크를 넣어도 더 정돈된 큐 동작
- 비과금 네트워크가 올 때까지 작업을 대기시키는 Wi-Fi only 흐름
- 다운로드 및 후처리 중 더 조심스러운 Android 임시 파일 처리
- 공격적인 병렬 작업에 약한 기기에서도 덜 부담스러운 UX

즉, 기능을 줄이는 것이 아니라, 예측 가능한 동작을 먼저 만드는 방향입니다.

## 기능

- [`yt-dlp`](https://github.com/yt-dlp/yt-dlp)가 지원하는 사이트의 영상/오디오 다운로드
- 큐 중심 작업 추가 및 관리 흐름
- 기기 부담을 줄이는 순차 다운로드
- 비과금 네트워크를 기준으로 동작하는 Wi-Fi only 다운로드 흐름
- 포맷 선택, 오디오 추출, 자막, 썸네일, 메타데이터 지원
- 고급 사용자를 위한 사용자 지정 `yt-dlp` 명령 템플릿
- Material 3 기반 안드로이드 UI

## 철학

Snetch는 모든 걸 한꺼번에 쏟아붓는 다운로더가 되려 하지 않습니다.

대신 이런 사용자에게 맞춰져 있습니다.

- 큐가 계속 이해 가능하길 원하는 사람
- 스마트폰과 태블릿에서 안정적으로 돌아가길 원하는 사람
- 백그라운드에서 무슨 일이 일어나는지 더 분명하길 원하는 사람
- 강력하지만 덜 혼란스러운 도구를 원하는 사람

이상적인 다운로드 경험이 "일단 줄 세우고, 차례대로 제대로 끝내자"에 가깝다면, Snetch의 방향도 그쪽입니다.

## 설치

릴리즈는 GitHub Releases를 통해 배포하는 것을 기준으로 합니다.

대부분의 최신 안드로이드 기기에는 아래 APK를 권장합니다.

- `arm64-v8a`

예정된 릴리즈 채널:

- 일상 사용용 안정 빌드
- 새 변경을 먼저 확인하는 프리뷰 빌드

## 상태

Snetch는 아직 적극적으로 분리 작업이 진행 중인 포크입니다. 일부 내부 구조와 이름, 소스 패키지는 아직 upstream 흔적을 남기고 있지만, 앱 설치 정체성과 공개 브랜딩은 별도 앱으로 정리해 나가고 있습니다.

즉, 현재는 다음을 예상하면 좋습니다.

- 브랜딩 정리 진행 중
- 패키지 및 릴리즈 다듬기 계속 진행 중
- 에뮬레이터와 실기기 양쪽에서 기능 검증 중

## 로드맵

- 포크 전용 브랜딩과 릴리즈 자산 정리 마무리
- 독립 배포를 위한 패키지 정체성 정리 확대
- 큐 가시성과 대기 상태 메시지 개선
- Android 저장소 및 후처리 동작 안정화 계속
- 더 깔끔한 공개 릴리즈 준비

## 크레딧

기반이 되는 오픈소스 프로젝트와 유지보수자들에게 공은 온전히 돌아가야 합니다.

이슈를 열 때는 아래 정보를 함께 적어주면 큰 도움이 됩니다.
